# LifeOS i18n Architecture

The Globalization Layer provides complete internationalization (i18n) support for LifeOS, enabling the system to serve users worldwide natively.

## Core Components

1. **Language Provider**
   The core engine responsible for translation resolution, interpolation, and namespace management. It follows the i18next specification structure.

2. **Translation Loader**
   Loads and caches JSON translation files from disk. Supports 8 native languages with 7 namespaces each.

3. **Locale Manager**
   Persists and retrieves user-specific locale configurations (language, timezone, currency) to ensure a consistent experience across sessions.

4. **Language Selector**
   Provides the API for UI components to list available languages, auto-detect the best language from browser tags, and dynamically switch the active locale.

5. **Locale Formatter**
   Powered by Babel, handles regional formatting for dates, times, currencies, and numbers according to the user's active locale.

6. **Companion i18n Bridge**
   Wraps the LifeOS Companion's AI outputs, ensuring all greetings, context messages, and decision recommendations are delivered in the user's language.

*Default Locale:* `pt_BR`
*Fallback Locale:* `en`

## Translation Architecture

Translations are stored in JSON format under `globalization_layer/locales/{locale_code}/{namespace}.json`.

**Namespaces:**
- `common`: Generic UI elements, navigation, actions, status
- `dashboard`: Dashboard widgets, missions, life score
- `companion`: Companion greetings, prompts, feedback
- `onboarding`: First-time setup flow
- `decisions`: Decision engine UI, categories, analysis
- `errors`: System and validation errors
- `formats`: Regional format masks

## Interpolation

The system supports variable interpolation using double braces `{{variable}}`:

```json
{
  "greeting": {
    "morning": "Good morning, {{name}}!"
  }
}
```

```python
provider.t("greeting.morning", name="Alex")
# Output: "Good morning, Alex!"
```
